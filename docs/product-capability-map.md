# Azurite Product Capability Map

## Purpose

This document preserves Azurite's full product ambition. It is a capability
universe rather than a backlog, release plan, architecture, or promise that
every surface must arrive at once.

The concise destination lives in the [product vision](product-vision.md).
Stable definitions for clusters, veins, lodes, context fabric, and Modules live
in the [product-language reference](reference/product-language.md).

## How To Read The Map

- Capabilities describe user outcomes. Architecture owns their mechanisms.
- The horizons describe increasing product completeness, not dates or delivery
  sequence.
- Obsidian and Notion are benchmark references. Azurite retains its own
  ontology, brand, formats, workflows, interfaces, and technical foundations.
- Markdown remains canonical for Azurite-authored knowledge. Original native
  files remain canonical source artifacts. Serious product state and derived
  intelligence may coexist under explicit ownership and portability promises.
- A capability may live in the core product or an Azurite Module. That boundary
  remains open where the mature experience does not yet require a decision.

## North-Star Product Test

> A person can begin meaningful work in any authorized surface with the right
> context already available, and trust that valuable outcomes will return to
> the right place with their provenance intact.

## Capability Horizons

### Sovereign Foundation

Azurite is an exceptional file-native knowledge environment for humans. It
provides complete knowledge lifecycle, Obsidian-level knowledge mechanics,
Notion-level interaction quality, original source-artifact support, dependable
recovery, and one coherent PWA experience.

### Connected Context

Clusters participate in lodes. Search, navigation, graphs, references,
automations, and Modules operate across deliberately authorized boundaries.
Context remains continuous across devices and product surfaces.

### Agent-Native Work

External harnesses and Azurite-native agents become first-class participants.
Sessions link to knowledge and work. Meaningful events can initiate authorized
activity. Agents return progress, blockers, results, and improved context to the
right clusters.

### Ambient Cognitive Environment

The context fabric stays current across life and work while authorized agents
and automations notice, connect, initiate, and return useful work. The person
gains compounding cognitive leverage without surrendering ownership, privacy,
or judgment.

## Representative Mature Experiences

### Code And LLMWiki

A Git repository is an Azurite Cluster. LLMWiki is another cluster. Authorized
cross-cluster veins let Codex, Claude Code, OpenCode, Pi, Hermes, OpenClaw, or a
future harness use accumulated methods, research, decisions, and prior work.
The active session stays linked to the files and project artifacts it concerns.
Its conclusions and history return to the appropriate cluster.

### Invoice To Budget Context

An invoice enters as an original PDF or image. The source remains intact.
Extraction, classification, metadata, synthesis, and citations become durable
Markdown knowledge. A recurring budget workflow receives the new context and
later analysis can trace every conclusion back to the source artifact.

### Agent-Driven Project Work

A Markdown-backed card moves into progress. The responsible external-agent
session is continued or created according to the user's automation rules. The
agent records progress, updates the card, links its session, and marks itself
blocked when human input is needed. The person can understand the current state
from Azurite without watching the agent continuously.

### Coaching Continuity

A chosen coaching or therapy-support persona receives only context the user has
authorized. It can draw on long-term reflections, goals, and prior sessions,
then preserve useful outcomes in the relevant cluster. Code, financial,
personal, and other context remain outside its scope unless they are both
relevant and explicitly authorized.

### Voice Across Devices

A person speaks into Azurite from a phone, tablet, or desktop. The production
host transcribes the recording into an editable note, card, command, or agent
message. The original recording can be retained when useful. An agent can return
text plus optional generated speech, allowing the person to listen instead of
reading. The same context and result remain available on every device.

### The Sunday Return

When the person returns after time away, Azurite presents an attributable
synthesis of what changed, what became connected, what agents or automations
initiated or completed, what remains blocked or contradictory, and what now
needs human judgment. Azurite coordinates and presents the context. The agents
and automations remain the explicit actors that noticed, reasoned, and acted.

## 1. File Sovereignty And Cluster Material

### Product Promise

Azurite increases the value of a person's files without making those files
dependent on Azurite.

### Capability Universe

- Adopt an existing notes folder, Git repository, project directory, synced
  folder, external drive, or other coherent file domain as a cluster.
- Create, identify, open, rename, move, copy, archive, detach, and deliberately
  remove clusters without confusing the cluster with the files it contains.
- Keep ordinary filesystem and native-tool workflows working when Azurite is
  absent.
- Treat Markdown as canonical authored knowledge, including notes, decisions,
  documentation, synthesis, metadata, and naturally Markdown-backed work.
- Treat code, PDFs, images, audio, video, datasets, scans, office documents, and
  other native files as canonical source artifacts for their own content.
- Preserve source artifacts while adding previews, extraction, OCR,
  transcription, annotation, metadata, summaries, and relationships.
- Keep generated context attributable to the exact source and version from
  which it came.
- Detect and reconcile changes made by editors, Git, scripts, sync tools,
  agents, and other external applications.
- Keep derived indexes, thumbnails, embeddings, caches, and previews separable
  from canonical material.
- Give durable product state an explicit ownership and portability contract when
  its loss would destroy user value.
- Support import, export, backup, restoration, and movement between hosted and
  self-hosted worlds without knowledge lock-in.

## 2. Complete Knowledge Lifecycle

### Product Promise

Knowledge can be captured, shaped, reorganized, recovered, and reused without
leaving broken references or silent loss behind.

### Capability Universe

- Create, read, edit, save, duplicate, rename, move, merge, split, archive,
  delete, trash, restore, and recover knowledge artifacts.
- Support folders, hierarchy, ordering, aliases, headings, blocks, lists,
  tables, tasks, code, mathematics, callouts, footnotes, and the selected
  Markdown dialect.
- Support frontmatter, typed properties, tags, dates, statuses, and structured
  metadata through approachable interactions.
- Create and use templates, recurring notes, daily notes, quick capture,
  snippets, commands, and bulk operations.
- Preserve links and references through rename and move operations.
- Surface malformed content, unresolved links, conflicts, external changes, and
  partial failures honestly.
- Provide version history, draft recovery, undo, comparison, and deliberate
  conflict resolution for human and agent edits.
- Keep original syntax inspectable and editable when a user wants source-level
  control.
- Support scriptable and automated knowledge maintenance without requiring the
  graphical interface.

## 3. Obsidian-Level Knowledge Capability

### Product Promise

Azurite matches the practical knowledge-management power that makes Obsidian
vaults exceptionally effective, then carries that power into Azurite's own
product world.

### Capability Universe

- Fast file and folder navigation, recent items, bookmarks, favorites, history,
  quick switching, commands, saved layouts, and contextual previews.
- Markdown links, wiki-style links, aliases, backlinks, outgoing links,
  unlinked mentions, embeds, transclusion, heading references, and block
  references.
- Tags, nested tags, properties, frontmatter, metadata exploration, queries,
  filtering, sorting, grouping, and structured views.
- Full-text, metadata, filename, path, property, scoped, and semantic search.
- Local graphs, cluster graphs, lode graphs, filtering, grouping, traversal,
  neighborhood exploration, and meaningful visual explanations.
- Unresolved-link discovery, broken-reference diagnostics, rename intelligence,
  reference lookup, document symbols, completions, and contextual hover
  information.
- Templates, recurring workflows, commands, automation, scripting, deep links,
  and programmatic access.
- Attachments, previews, embeds, audio capture, source-file navigation, and rich
  views over file-owned material.
- Visual and structured knowledge experiences with Azurite-owned portable
  representations and clear source-of-truth rules.
- Recovery, synchronization, import, export, publishing, and collaboration
  capabilities where they belong in Azurite's product horizon.
- Extensibility sufficient for users and third parties to build important
  knowledge workflows without waiting for the core product.

### Parity Discipline

The parity target requires a researched inventory of Obsidian core behavior,
first-party capabilities, important community workflows, and user scripting.
A capability is not dismissed merely because Obsidian delivers it through an
extension. Each capability is assigned deliberately to Azurite core, an
official Module, the wider Module ecosystem, or an explicit departure.

Functional parity means equivalent user outcomes and programmatic power. It
does not require Obsidian's interface, terminology, vault metadata, file
markers, or implementation architecture.

## 4. Veins, Lodes, And Cross-Cluster Knowledge

### Product Promise

Knowledge can cross domain boundaries without collapsing ownership,
provenance, or authorization.

### Capability Universe

- Use typed veins for authored links, references, embeds, shared metadata,
  source relationships, agent-session associations, observed activity, and
  explainable semantic inference.
- Preserve each vein's origin, actor, direction, certainty, scope, and source.
- Distinguish relationship origin and certainty from relationship lifecycle.
  Authored, observed, and inferred describe how a relationship arose;
  candidate, proposed, and materialized describe its current standing.
- Keep candidate or inferred relationships from creating lode topology until
  they are deliberately materialized under the user's rules.
- Navigate, search, cite, preview, and traverse across authorized cluster
  boundaries.
- Explore local, cluster, lode, and account-level graph views without implying
  that visibility grants content access.
- Assemble task-specific context from several authorized clusters while keeping
  every source traceable.
- Let lodes emerge from materialized cross-cluster veins and support future
  named or deliberately scoped lode experiences.
- Preserve participating clusters as independently usable domains.
- Make relationship visibility and content authorization separate controls.

## 5. Notion-Level Human Experience

### Product Promise

Azurite feels calm, immediate, beautiful, and obvious while retaining the power
and inspectability of files.

### Capability Universe

- A fluent Markdown-backed editor with rich direct manipulation and faithful
  round trips.
- Approachable slash commands, formatting controls, drag and drop, reordering,
  nesting, selection, quick insertion, and keyboard-first operation.
- Clear file exploration, tabs, panes, breadcrumbs, history, previews,
  sidebars, quick switching, commands, and focused writing modes.
- Rich interaction with properties, tags, references, tasks, source artifacts,
  embeds, graphs, and structured views.
- Serious desktop depth and a mobile experience designed for touch, capture,
  reading, review, editing, and agent interaction.
- Surface-appropriate tablet and watch-class experiences where PWA platforms
  permit useful interaction.
- Smooth transitions, immediate feedback, understandable progress, and graceful
  recovery from interruption or reconnect.
- Accessibility across keyboard, touch, screen readers, reduced motion,
  contrast needs, and assistive technologies.
- A distinctive Azurite visual identity and a coherent theming system without
  generic productivity-app sludge.
- Instant-feeling behavior across large clusters and lodes.

## 6. Multimodal Human Interaction

### Product Promise

People can capture, understand, and receive knowledge through the mode that
fits their situation.

### Capability Universe

- Record speech from any supported PWA surface and use it as note content,
  search, a command, project input, or an agent message.
- Provide accurate multilingual speech-to-text with editable transcripts,
  punctuation, timestamps, speaker context where available, and source
  provenance.
- Never let transcription silently replace or destroy an original recording.
  Let users decide whether it is retained, attached, or deliberately discarded.
- Generate optional text-to-speech for agent replies, documents, summaries,
  reviews, and other chosen knowledge.
- Let generated audio remain ephemeral or become a first-class source artifact
  linked to its text and generating session.
- Provide playback speed, seeking, pause and resume, queueing, and clear
  synchronization between text and audio where useful.
- Allow conversational voice interaction without requiring every agent workflow
  to become a voice assistant.
- Support local, self-hosted, and future provider-backed speech models under the
  same product experience.

Current dogfooding context: Daniel's production host already has NVIDIA
Parakeet transcription and Qwen3 TTS available. They provide the first path for
testing this product promise while remaining replaceable model choices.

## 7. Continuous Context Fabric

### Product Promise

Relevant context stays current and ready while canonical material remains
owned, attributable, and inspectable.

### Capability Universe

- Continuously discover and refresh files, metadata, relationships, active
  work, linked sessions, source artifacts, and authorized external updates.
- Assemble relevant context around a task, file, person, persona, project,
  question, or conversation.
- Preserve provenance from source artifact through extraction, synthesis,
  inference, action, and writeback.
- Track freshness, staleness, contradiction, confidence, and failed updates.
- Return valuable outcomes to their appropriate clusters without creating
  unsorted AI debris.
- Keep context useful across model changes, provider changes, harness changes,
  device changes, and long periods of time.
- Present a return synthesis of changes, new connections, initiated and
  completed work, blockers, contradictions, and decisions that need human
  judgment.
- Offer context inspection so users can understand why something was included,
  excluded, connected, or suggested.
- Let users shape relevance, boundaries, retention, and automatic maintenance.
- Preserve the distinction that Azurite maintains and routes the fabric while
  agents and automations notice, reason, decide, and act.

## 8. External-Agent-Primary Participation

### Product Promise

Azurite delivers its full contextual value even during weeks when a person
rarely opens the Azurite interface.

### Capability Universe

- Let Codex, Claude Code, OpenCode, Pi, Hermes, OpenClaw, ChatGPT surfaces,
  API-backed agents, and future harnesses participate under explicit authority.
- Give authorized agents open, stable ways to discover, query, search, cite,
  create, edit, move, connect, and update Azurite material.
- Treat a CLI and agent skill as preferred broad product surfaces while allowing
  APIs, MCP, deep links, provider-specific integrations, and future transports
  where they improve the experience.
- Make code repositories effective clusters without compromising their native
  Git and development workflows.
- Assemble authorized context from LLMWiki, project clusters, personal
  knowledge, and Module-owned worlds according to the current task.
- Return valuable agent outcomes to canonical files or explicitly owned durable
  state with provenance intact.
- Avoid provider lock-in by keeping the product promise harness-agnostic.
- Let humans inspect and control which agents can read, write, act, subscribe,
  or cross cluster boundaries.

## 9. Linked Sessions And Bidirectional Work

### Product Promise

Conversations and agent sessions remain attached to the work they advance
instead of becoming disposable islands.

### Capability Universe

- Link external or Azurite-native sessions to files, headings, blocks, cards,
  decisions, questions, projects, clusters, and lodes.
- Continue an appropriate existing session or create a new one according to
  explicit workflow rules.
- Notify relevant sessions when authorized Azurite material changes.
- Let sessions report progress, blockers, decisions, outputs, and completion
  back to Azurite.
- Preserve actor, harness, model, persona, source context, timestamps, and
  relationship history where valuable.
- Make session summaries and durable outcomes accessible without requiring the
  original harness to remain available forever.
- Support interruption, redirection, cancellation, retry, reassignment, and
  human takeover.
- Coordinate and hand work among authorized external and Azurite-native
  sessions while preserving actor, source, status, and provenance.
- Prevent noisy event loops, duplicate action, and invisible autonomous churn
  as product-level trust requirements.

## 10. Event-Driven Knowledge And Work

### Product Promise

Meaningful changes can initiate useful authorized work, and every action leaves
an understandable trail.

### Capability Universe

- Trigger work from file changes, card transitions, new source artifacts,
  schedules, recurring reviews, external automations, and explicit commands.
- Route each event to the right agent, automation, Module, or human attention
  according to visible rules.
- Support approval-required, supervised, and autonomous modes per workflow.
- Record trigger, actor, inputs, status, outputs, error, blocker, and resulting
  knowledge changes.
- Keep automated work idempotent and understandable from the human product
  surface.
- Let agents update workflow state when they begin, progress, block, complete,
  or require a decision.
- Make recurring research feeds, scans, invoices, budget reviews, and other life
  automations part of the same context fabric.

## 11. Azurite-Native Agent Experiences

### Product Promise

People can add focused agent environments inside Azurite while the core context
fabric remains useful without a mandatory assistant persona.

### Capability Universe

- An installable Agent Module that turns Azurite into an agent harness.
- Supported subscription-backed providers where their platforms permit it,
  user-supplied API providers, local models, and future provider adapters.
- Persistent agents, selectable models, personas, tools, source grounding,
  context scopes, and durable writeback.
- Research, synthesis, coding, planning, reflection, coaching, and specialist
  workflows.
- Agent sessions that participate in veins and lodes like other knowledge
  artifacts.
- Inspectable activity, permissions, budgets, schedules, interruption, and
  redirection.
- Multiple specialist agents that can cooperate without imposing a universal
  Azurite personality.
- Persona definitions that remain user-owned, portable, and separable from the
  model provider.

## 12. Modules And Extensible Product Worlds

### Product Promise

Azurite can expand into specialized work without bloating or fragmenting its
core experience.

### Capability Universe

- Install, enable, disable, configure, update, inspect, and remove Azurite
  Modules.
- Let Modules add interfaces, views, commands, workflows, artifact types,
  knowledge behavior, agents, automation, and external integrations.
- Give every Module clear permissions, cluster scope, network authority, agent
  authority, and durable-state ownership.
- Keep Module-created knowledge file-sovereign and Markdown-backed where the
  artifact is naturally knowledge or work content.
- Allow rich Module state where the product truth requires it, with explicit
  export, recovery, and portability expectations.
- Support official, community, private, and organization-specific Modules.
- Provide discovery, documentation, trust signals, compatibility information,
  and future ecosystem distribution.
- Keep product Modules linguistically distinct from code packages, components,
  services, and subsystems.

### Candidate Modules

- Project management and work coordination.
- Azurite-native agent harness.
- Codebase intelligence and repository context.
- Document ingestion, OCR, transcription, and research synthesis.
- Coaching and therapy-support environments.
- Budgeting, invoices, recurring reviews, and personal operations.
- Publishing, collaboration, presentation, and domain-specific knowledge views.

## 13. Markdown-Backed Project Operations

### Product Promise

Human and agent work can be coordinated from durable project knowledge rather
than a disconnected task database.

### Capability Universe

- Markdown-backed cards and project records with board, list, timeline,
  calendar, filtered, and focused views.
- Relationships from cards to specifications, source files, code, decisions,
  agents, sessions, outputs, and blockers.
- Status, priority, owner, agent, dates, dependencies, acceptance evidence, and
  history that remain intelligible outside the visual board.
- Human and agent movement of work through states under visible authority.
- Automatic session continuation or creation when work enters an actionable
  state.
- Agent-maintained progress, blocker, result, and verification information.
- Clear distinction between intended work, active work, waiting, blocked,
  review, and completed outcomes.
- Project views that remain useful for purely human work as well as agent-heavy
  operation.

## 14. One Product Across Devices

### Product Promise

The human experience simply continues wherever the person is.

### Capability Universe

- The same account, clusters, lodes, knowledge, permissions, context, drafts,
  sessions, and work state across desktop, phone, tablet, and future useful
  PWA-capable surfaces.
- No product-visible export, import, or handoff ritual between devices.
- Deep creation and multi-pane work on desktop.
- Fluid capture, review, editing, voice, notifications, and agent interaction on
  mobile.
- Touch-appropriate tablet workflows and focused watch-class capture or review
  where platform capabilities permit them.
- Graceful offline use, reload, app switching, tab discard, reconnect, and
  continuation.
- Notifications and resumable deep links that return the person to the exact
  relevant context.
- Device-specific presentation without fragmenting the product into separate
  applications or knowledge worlds.

## 15. Accounts, Editions, And Distribution

### Product Promise

Azurite can be self-owned, effortlessly hosted, and continuously dogfooded
without changing the user's relationship to their knowledge.

### Capability Universe

- A polished open-source self-hosted edition suitable for serious daily use.
- A hosted production edition with account login, cross-device continuity,
  operational simplicity, and future subscription value.
- Effectively unlimited clusters per account, bounded by chosen storage and
  service realities rather than artificial organization limits.
- Clear movement, backup, export, and restoration between hosted and self-hosted
  operation.
- Private-by-default local and Tailscale-oriented use.
- Deliberate sharing or public exposure when the product later supports it.
- A production runtime independent from the source-development checkout.
- Daniel's own daily Azurite running through the real production distribution.
- Transparent edition boundaries that preserve foundational sovereignty and
  avoid holding user knowledge hostage to a subscription.

## 16. Collaboration And Shared Context

### Product Promise

When knowledge becomes shared, its ownership, attribution, and agent activity
remain understandable.

### Capability Universe

- Shared clusters, deliberately shared lode experiences, and scoped external
  participation.
- People, teams, guests, agents, Modules, and automations with explicit roles
  and permissions.
- Attribution, comments, review, suggestions, presence, and shared activity
  history where they improve the workflow.
- Conflict handling for simultaneous human, agent, filesystem, and automation
  changes.
- Shared agents and automations whose authority is visible to every affected
  participant.
- File sovereignty and exportability preserved when knowledge becomes
  collaborative.
- Clear boundaries between personal, shared, organization, and published
  context.

Collaboration remains a product frontier rather than an assumption that every
cluster is collaborative.

## 17. Trust, Authorization, Provenance, And Control

### Product Promise

Power remains legible. Users know what Azurite and its participants can access,
what happened, and how to recover.

### Capability Universe

- Per-cluster and cross-cluster authority for people, agents, Modules,
  automations, devices, and external services.
- A vein or lode membership never granting content access automatically.
- Clear separation of authored fact, observed fact, inference, suggestion, and
  generated synthesis.
- Provenance for automated edits, extractions, relationships, summaries,
  decisions, and actions.
- Human-configurable approval, supervision, autonomy, and writeback levels.
- Reversible agent and automation actions wherever the underlying operation can
  be reversed.
- Visible conflicts, stale context, failed extraction, partial updates, and
  blocked work.
- No silent data loss, context corruption, privilege expansion, or promotion of
  inference into authored truth.
- Deliberate containment of personal, coaching, therapy-support, financial,
  project, and code contexts.
- Credential containment without censoring the user's own product data from
  user-controlled debugging and observability.
- Activity and audit views useful to ordinary humans rather than only to system
  operators.

## 18. Performance, Dependability, And Pleasure

### Product Promise

Azurite earns daily trust through speed, resilience, clarity, and delight.

### Capability Universe

- Instant-feeling capture, editing, navigation, search, linking, graph
  exploration, and context assembly.
- Predictable performance across large clusters, lodes, source collections, and
  long histories.
- Dependable save, synchronization, recovery, conflict handling, and external
  file reconciliation.
- Fresh indexes and explainable degradation when context is incomplete or
  rebuilding.
- Robust behavior through reloads, reconnects, mobile lifecycle interruption,

- Accessible, responsive, visually coherent interaction across supported

- Calm defaults, progressive disclosure, and powerful expert paths.
- Observability sufficient to make real failures understandable without making

- A level of polish that makes Azurite pleasurable enough to become the place a
  person wants their knowledge and work to live.

## Deliberate Departures From Benchmarks

### From Obsidian

- Azurite uses clusters, veins, lodes, and Modules as its own product language.
- Cross-cluster context and lode exploration are first-class product territory.
- External and embedded agents are participants in the knowledge model rather
  than chat windows attached to it.
- One PWA delivers the product across devices and deployment choices.
- Obsidian compatibility may occur naturally through ordinary files, but it is
  not a governing constraint.

### From Notion

- Files remain user-owned and independently useful.
- Markdown remains canonical for authored knowledge rather than serving as an
  export approximation.
- Rich interaction does not require adopting Notion's block ontology or
  proprietary content model.
- Local, self-hosted, Git-based, scripted, and agent-based workflows are core
  product realities.

### Azurite's Own Territory

- Continuous context across clusters, lodes, devices, agents, automations, and
  time.
- Bidirectional work between durable knowledge and external agent sessions.
- Original source artifacts, authored knowledge, durable product state, and
  derived intelligence living together under explicit truth rules.
- A geological knowledge world where files form clusters, veins connect
  knowledge, and connected clusters form lodes.

## Open Product Frontiers

- The exact researched boundary of Obsidian parity across core, first-party,
  community, and scripted workflows.
- Whether ordinary Markdown interchange is sufficient or selected Obsidian
  import and export behavior should become a deliberate capability.
- Cluster identity and the user experience for nested, overlapping, copied, and
  remotely hosted cluster roots.
- Named, curated, task-scoped, and derived lode experiences beyond the settled
  geological vocabulary.
- Which durable product truths require portable file representation, structured
  export, or synchronized application state.
- The exact boundary between universal capabilities and Azurite Modules.
- The autonomy policies for thread creation, event delivery, canonical
  writeback, project-state changes, and multi-agent coordination.
- The capability relationship between self-hosted and hosted editions.
- The business model, subscription value, Module ecosystem, and commercial
  distribution.
- The product and safety boundary for coaching and therapy-support Modules.
- The collaboration model for personal, shared, organization, and published
  knowledge.
- Which watch-class and background experiences remain reliable across actual
  PWA platform constraints.
