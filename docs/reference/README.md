# Azurite Reference

This folder tracks stable product and system details that should be easy to
reuse across implementation slices.

Use reference docs for facts that code, tests, docs, and future UI work should
name consistently, such as API error codes, route shapes, reserved workspace
names, and runtime messages.

Current reference pages:

- `api-errors.md`
- `api-routes.md`
- `product-guardrails.md`
- `runtime-messages.md`
- `workspace-rules.md`

Use slice docs for step-by-step implementation plans and completion records.
Use a future decision-record folder only when the project needs to preserve the
reasoning behind a larger architectural or product choice.
